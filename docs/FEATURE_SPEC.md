# 기능 명세

## 1. 메인 검색 (MVP 핵심)

### 1.1 출발지/도착지 입력
- 주소, 역명, 지명 자동완성 검색
- Kakao Local API 활용
- 최근 검색 기록 (localStorage)

### 1.2 대중교통 경로 탐색
- Kakao 대중교통 경로 API 호출
- 지하철 + 버스 경로 모두 지원
- 경로상 정류장/역 좌표 목록 추출

### 1.3 경로 기반 매물 검색
- 각 정류장/역에서 반경 N미터 내 공공임대 검색
- PostGIS `ST_DWithin` 사용
- 반경 옵션: 도보 5분(400m) / 10분(800m) / 15분(1.2km)

## 2. 필터링 (드롭다운 + 슬라이더)

### 2.1 대상 유형 필터 (드롭다운, 필수)
```
[전체] ▼
├── 청년        (만 19~39세, 미혼, 소득 기준 충족)
├── 신혼부부    (혼인기간 7년 이내 또는 예비신혼)
├── 일반        (무주택세대구성원)
├── 대학생      (대학 재학 또는 입학예정)
├── 고령자      (만 65세 이상)
└── 주거급여수급자
```
- 선택 시 해당 대상에 맞는 자격요건이 함께 표시됨
- `data/housing/eligibility.json` 에서 로드

### 2.2 공급유형 필터 (드롭다운, 복수 선택)
```
[전체] ▼
├── 행복주택
├── 국민임대
├── 영구임대
├── 매입임대
├── 전세임대
└── 공공지원민간임대
```

### 2.3 보증금 범위 필터 (이중 슬라이더)
- 최소 보증금 ~ 최대 보증금 (단위: 만원)
- 범위: 0 ~ 30,000만원
- 스텝: 100만원

### 2.4 보증금 전환 (최소전환 / 최대전환)
행복주택 등은 보증금↔월세 전환이 가능함.
- **최소전환**: 보증금을 최소로 낮추고 월세를 올린 금액
- **최대전환**: 보증금을 최대로 올리고 월세를 낮춘 금액
- 전환율: `data/housing/deposit_table.json` 에서 로드
- UI: 각 매물 카드에 "보증금 최소전환 / 최대전환" 컬럼 표시

### 2.5 모집상태 필터
```
[전체] ▼
├── 현재 모집중
├── 모집 예정
├── 마감
└── 과거 이력
```

## 3. 정렬

### 3.1 정렬 옵션 (드롭다운)
```
[정렬 기준] ▼
├── 보증금 낮은순 (기본값)
├── 보증금 높은순
├── 최소전환 보증금 낮은순

├── 월임대료 낮은순
├── 월임대료 높은순
├── 전용면적 넓은순
├── 전용면적 좁은순

```

## 4. 검색 결과 화면

### 4.1 지도 뷰
- 대중교통 경로 폴리라인 표시
- 매물 위치 핀 마커 (유형별 색상 구분)
- 핀 클릭 → 간단 정보 인포윈도우
- 클러스터링 (매물 밀집 시)

### 4.2 리스트 뷰
각 매물 카드 구성:
```
┌─────────────────────────────────────────────────┐
│ [행복주택] 단지명                    [모집중] 🔴  │
│ 서울특별시 강남구 역삼동 123-45                    │
│─────────────────────────────────────────────────│
│ 전용면적: 26㎡  |  공급세대: 150세대               │
│─────────────────────────────────────────────────│
│ 보증금      월임대료     최소전환     최대전환       │
│ 2,800만원   15만원    1,200만원   5,400만원       │
│─────────────────────────────────────────────────│
│ 대상: 청년, 신혼부부  |  마감: 2026.04.15          │
│ 📍 역삼역 도보 8분                                │
└─────────────────────────────────────────────────┘
```

### 4.3 레이아웃 (반응형)
| 화면 | 구성 |
|------|------|
| Desktop 1024px+ | 좌측 리스트(40%) + 우측 지도(60%) |
| Tablet 768~1023px | 상단 지도(50vh) + 하단 리스트 |
| Mobile ~767px | 탭 전환(지도/리스트) + 바텀시트 상세 |

## 5. 상세 페이지

### 5.1 단지 상세
- 단지 기본 정보 (이름, 주소, 유형, 세대수, 준공일)
- 지도 (단지 위치 표시)
- 임대 조건 테이블 (면적별 보증금/월세/최소전환/최대전환)
- 자격요건 카드 (대상별 소득/자산/연령/거주지 요건)
- 모집 이력 타임라인 (과거 공고 목록)
- 원본 공고 링크

### 5.2 자격요건 상세
대상 유형별 탭으로 표시:
```
[청년] [신혼부부] [일반] [대학생] [고령자]
──────────────────────────────────────
소득기준: 도시근로자 월평균소득 100% 이하
자산기준: 총자산 3.61억 이하, 자동차 3,683만원 이하
연령기준: 만 19세 ~ 39세
거주요건: 해당 지역 거주자 또는 인근 지역 거주자
우선순위: 해당 지역 거주 > 인근 지역 거주
```

## 6. 데이터 로딩 전략

### 6.1 정적 데이터 (JSON 파일 → 메모리)
변경이 거의 없는 데이터는 JSON 파일로 관리:
- `eligibility.json` — 대상별 자격요건
- `deposit_table.json` — 보증금-월세 전환율 테이블
- `categories.json` — 공급유형/대상유형/모집상태 카테고리

Django AppConfig.ready()에서 로드하거나,
서비스 레이어에서 `@lru_cache`로 캐싱.

### 6.2 동적 데이터 (크롤링 → PostgreSQL)
주기적으로 변경되는 데이터:
- 모집공고 (신규/마감/상태변경)
- 단지 정보 (신규 추가)
- 공급세대 정보
