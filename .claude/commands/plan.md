# 기능 기획 분석

사용자가 요청한 기능에 대해 아래 프로세스를 따라 분석하고 구현 계획을 세워주세요.

## 1단계: 요구사항 분석
- docs/FEATURE_SPEC.md 를 읽고 기존 기능 명세 확인
- docs/ARCHITECTURE.md 를 읽고 아키텍처 제약사항 확인
- 요청된 기능이 기존 명세에 있는지 / 신규인지 판별

## 2단계: 영향 분석
- 변경이 필요한 파일 목록 작성 (Backend + Frontend)
- 새로 생성할 파일 목록 작성
- DB 마이그레이션 필요 여부 판별
- JSON 정적 데이터 변경 필요 여부 판별

## 3단계: 구현 계획
아래 형식으로 출력:

```
## 구현 계획: [기능명]

### 백엔드 변경
- [ ] 파일: 변경 내용 요약
...

### 프론트엔드 변경
- [ ] 파일: 변경 내용 요약
...

### 데이터 변경
- [ ] 파일: 변경 내용 요약
...

### 테스트
- [ ] 테스트 항목
...

### 예상 소요: N시간
```

사용자 요청: $ARGUMENTS
